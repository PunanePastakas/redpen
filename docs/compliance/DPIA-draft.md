### 1.2 Functional Description

*Provide a step-by-step description of the data flow:*

1. Teacher creates a RedPen account (email, name, school)
2. Teacher photographs pupil homework using mobile device
3. RedPen app strips EXIF metadata from the image
4. Image is uploaded to RedPen's EU-hosted servers
5. RedPen sends the image to OpenAI EU API endpoint (`eu.api.openai.com`) with a grading prompt
6. OpenAI processes the image in-memory within EU infrastructure, returns grading suggestion — zero data retention, image is not stored by OpenAI
7. RedPen receives the AI response and presents it to the teacher with confidence score
8. Teacher reviews, modifies if needed, and confirms the grade
9. RedPen stores: the original image, AI suggestion, teacher's final decision, timestamp, model version — in EU-hosted database
10. Audit log entry is created for compliance purposes
11. Data is retained for [RETENTION PERIOD] and then automatically deleted

### 2.2 Special Category Data (Article 9)

*Address handwriting as potential biometric data:*

- **Analysis:** Handwriting can constitute biometric data under Art. 4(14) if processed for the purpose of uniquely identifying a natural person. RedPen processes handwriting solely for content recognition (grading), not for identification.
- **Conclusion:** *Document the reasoning that Art. 9 is not triggered because the processing purpose is content extraction, not biometric identification. Reference the zero-retention architecture as a safeguard against repurposing.*
- **Safeguard:** *Confirm that prompts to OpenAI never request handwriting analysis, style comparison, or identification.*

### 2.3 Children's Data (Article 8)

- **Estonian consent age:** 13 years (Personal Data Protection Act, Section 8(1))
- **For pupils aged 13+:** *Document that the service is directed to teachers, not to children. The children are data subjects but not users of an information society service. Article 8 consent requirements for information society services may not apply directly — document the reasoning.*
- **For pupils under 13:** *Parental consent or an alternative legal basis (public task via school) is required. Document the approach.*
- **Safeguards regardless of age:** School notification, parent notification, data minimization, human oversight of all AI outputs
